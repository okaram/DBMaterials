Connecting with a slow network

In order to practice SQL in the server and to complete the final project, there are three kinds of things you need to do in the server:

  1. Execute programs on the server, like psql
  2. Create and change text (html,php,sql) files on the server
  3. (maybe) Transfer files created elsewhere (images etc) into the server

This document will show you how to do so in a MS Windows machine (XP, Vista or 7).

Execute programs on the server

In order to connect to the server to execute programs or commands on it, you need an SSH client with a terminal emulator; a simple one is putty, which you can download from here.

After downloading putty, run it (it's just a plain executable) and configure it as in the image below, making sure:

  • The hostname is okaram.spsu.edu
  • The port is 22 (that is the default)
  • The connection type is set to SSH (default)

Later, you may want to save your session (or session settings) and give it a name. If you don't like the font etc, they can be changed in the appearance tab. Also, if you write myusername@okaram.spsu.edu (replacing with your actual username) as the server, it won't ask you your username again

The first time you log in with putty, it will ask you whether you're connecting to the right server; unless you're paranoid, just say yes :) (the risk is a man-in-the-middle attack, where some server could be maskerading as okaram.spsu.edu, and messing with your DNS server; possible, but not very likely :). The dialog is illustrated below

After that, it will ask you for your username (same as your spsu email) and password (the initial one has been provided by email and is also listed on our WebCT site for the online section); the final result should be a screen like the one below, where you can start typing commands (like psql). Remember that the prompt changes between psql and bash, \q gets you out of psql, and exit logs you out from the server nicely, although chances are nothing bad will happen if you just close putty)

Creating and changing text (html,php) files on the server

You could run a text editor on the server to create and edit files (vi, emacs or pico if you're only using putty, bluefish and emacs if you're also using an Xwindows server, like with XLiveCD), however, if the network connection is slow, it may be a painful experience. Here we describe how to connect using Komodo edit, a free (no cost) editor that allows you to save files remotely using sftp (BTW, komodo edit is multuplatform, so you can also get it for Mac and Linux, although many basic text editors on Linux also support this feature). If you prefer, you may want to check this Youtube video, or you may keep reading for a reference.

Once you've downloaded the program, run it and select (on the menus) File, then Open, then Remote File, and you get a dialog like the one below (but I've added the red circle and arrow to tell you where to click :)

Then click on the icon at the top-right corner, at the end of the line that begins with server (see the red circle and arrow on the picture above :) and you get another dialog like the one below (Note: you could also get to this same dialog by going to edit / preferences on the menus)

Make sure:

  • The server type is SFTP
  • The name is whichever name you want to give to this connection setting (I call mine ok)
  • The hostname is okaram.spsu.edu
  • Port is blank
  • Username and password are your Linux username and password
  • All other values are as shown

After you click on OK, you get back to the same file open dialog, but now you can select your server from the server drop down, as shown below (and this setting should be stored so you can always just pick that server). Notice it may take a little for the dialog to reflect the information on the server.

You can use a similar procedure to save a previously open file remotely.
