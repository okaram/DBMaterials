Connecting to the server through a slow network

Editing files on the server over a slow network is painful (since most keystrokes involve a trip to the server to tell the program running there, and a trip back to tell your screen to change); so I've found a text editor that runs on *your* machine, but still allows you to open and save files on the server (and that's free and multiplatform :); so I've now written a new writeup (Connecting with a slow network). By the way, this also works if you don't like XLiveCD or Putty and XMing.

You may also want to check the following Video on accessing the server with Komodo

Other ways to connect to the server

  • Using XliveCD from Home XLiveCD has all you need, you do not install anything on your machine, but only works for Windows XP. You can also check in this Connecting with XliveCD video or, if you're in our labs this Connecting from our labs video
  • If you're using Windows Vista or 7, you probably want to follow the instructions at Putty+Xming notes and this Putty+Xming video
  • A quick way to connect from Windows, but that does NOT allow you to run GUI programs like buefish or mrxvt is just downloading putty as described in Connecting with putty only
  • If you use Linux, you probably have an XWindows server and ssh, so just start from ssh
  • If you use Mac OSX, you probably have ssh but may not have XWindows. You can get XWindows from X11 for MacOS X, and then open a terminal window and start from ssh ...
